Review

The above examples illustrate the following points:

  • JavaScript code can be the contents of a <script> element or the value of an event attribute.
  • Use document.write() to write HTML source into a document which the browser then interprets.
  • Statements are terminated with a semicolon.
  • Use the onlick, onmouseover, and onblur attributes to specify event handlers.
  • For each HTML element in a web page the browser creates an object. For each attribute, the object has a property with the same name.
  • this always refers to the current object, such as the textbox for which we are coding an onclick event handler.
  • Strings are delimited with either single or double quotes.
  • The alert() method pops up a dialog box.
  • A block is a set of statements enclosed in curly braces. For example,
    {var insult = "You're a blockhead, Charlie Brown!"; alert(insult);}
  • Use an if statement to conditionally execute a block. For example,
    if (this.value === 'Charlie Brown') {alert("You're a blockhead!");}

Express Yourself

Our goal in this chapter is to teach you what could be called "conversational JavaScript" by providing a set of examples that you can use as models for doing basic tasks like getting input, displaying output, doing basic math, responding to mouse clicks and mouseovers, etc. We'd like you to start speaking JavaScript right away without having to first spend a lot of time studying the underlying structure and syntactic details of the language.

So, here are a few more (not as simple) examples that show how to do some useful tasks. They illustrate creating variables for storing values, gluing strings together, doing basic arithmetic, and comparing values.

Review

The examples in this section demonstrate these additional concepts, language elements, and techniques:

  • A value can be a number, string, boolean, array, function, object, or the special value undefined.
  • // initiates a single line comment.
  • The var statement is used to declare a set of variables each of which may be assigned an initial value.
  • An array can be created with an array literal such as ['Corvette', 'Cadillac', 'Impala'].
  • A function literal has the form function() block. For example,
    function() {alert('I just flew in from New York. And boy are my arms tired.');}
  • An object can be created with an object literal such as
    {name: 'Jack', age: 39, occupation: 'comedian'}    .
  • If a variable has not been given a value, it has the value undefined.
  • document.writeln() writes its arguments to the document followed by a newline character.
  • The escape sequence \n inside a string represents the newline character.
  • To call a function, append () to the function name.
  • To glue two strings together, use the + operator. If only one of the operands is a string, the other will be converted to a string and glued on.
  • document.getElementById() is a great way to get the object for any element in the document. Be sure the browser has created the element and its object before trying to get it. The browser processes the HTML document from top to bottom. Functions and event handlers do not run until they are called.
  • The contents of a textbox is a string. If you want to do math with it, convert it to a number with either parseInt() or parseFloat().
  • We can compare two values for equality with ===, inequality with !==, less than with <, less than or equal to with <=, greater than with >, and greater than or equal to with >=.

OK, you've been reading JavaScript, now it's time to practice speaking it.

Go With the Flow

The browser processes the HTML file from top to bottom. However, only one of the blocks in an if else statement gets executed. And by using a loop we can cause a block to be executed multiple times. Also, functions and event handlers do not actually run until they are called. The following four scripts exemplify these control flow concepts of selection, repetition, and invocation.

Review

In the last four examples we have seen how to control the program flow by selecting a block with if else, repeating a block with either for in or while, and calling a function. Here are the new things we have seen demonstrated:

  • The if else statement selects one block or another for execution based on the truth value (true or false) of a condition. For example,
    if (likeCircus) { // true block
       runAwayWithCircus();
    }     else { // false block
       goToSchool();
    }
  • The screen object contains information about the user's screen such as width, height, and colorDepth.
  • The for in statement iterates through all the properties of an object. Each property has a name and a value. The block, called the body of the loop, gets repeated for each property. For example,
    var willie = {firstName: 'Emmett', lastName: 'Kelly', clownName: 'Weary Willie'};
    for (prop in willie) {
       alert(willie[prop]);
    }
    This snippet of code pops up three alerts. The first says Emmett, the second Kelly, and the third Weary Willie.
  • The while statement loops as long as the condition is true. For example,
    while (true) { // loop forever (probably not a good idea)
       alert('Number 9');
    }
  • The confirm() method pops up a dialog box with OK and Cancel buttons. It returns a value of true or false. Guess which button returns what value.
  • An event handler is often a function call. First we put the function definition in the <head> element. For example,
    var enlargeImage = function (img) {
       img.width *= 2;
       img.height *= 2;
    }     // enlargeImage
    Then we can make a function call from anywhere in the page. For example,
    <img onmouseover="enlargeImage(this);" ...
    When the image is moused over the enlargeImage() function is called. The argument this is passed to the receiving parameter img. Since the image to enlarge is passed to function, this function can be used to enlarge any image in the page.
  • ! is the logical not operator. It flips the truth value of its operand.

Review

In this section we introduced three more event attributes.

  • We have now seen examples of six event attributes: onclick, onmouseover, onmouseout, onfocus, onblur, and onload.
  • An element fires an onmouseout event when the mouse moves off of it.
  • An element fires an onfocus event when it gets the focus.
  • The <body> element fires an onload event when the page has finished loading.
  • An <img> element fires an onload event when the image has finished loading.
  • If the event handling code can easily fit on one line, you can put it inline as the attribute value.
  • If the event handler is more than one line, you must use a function call.
  • Whenever a significant amount of code is needed in more than one place, put the code in a function in the <head> and call the function when needed. This is known as centralized maintenance and code reuse - both very desirable features.
  • We can pass a value, such as an image or a string, to each parameter of the function for it to use. This makes the function more flexible and powerful.
  • Functions and event handlers run when called. This has nothing to do with the order in which they appear in the source code. Put function definitions in the <head> element. An event handler is the value of an event attribute.

Alley OOP

JavaScript is an Object-Oriented Programming (OOP) language. We have seen many objects already in the previous examples: document, screen, image objects, textbox objects, arrays, functions, and object literals. Objects have properties such as write, width, height, value, getElementById, src, and colorDepth. If the value of a property is a function, such as is the case for the document object's write and getElementById properties, we call it a method.
