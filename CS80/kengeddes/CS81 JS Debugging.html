What is Debugging?

  • Whenever you write a script or program, you will inevitably make mistakes.
  • debugging - finding and eliminating coding errors (aka bugs) in a script or program
  • debugger - a program that helps a programmer find and eradicate bugs in a computer program
  • types of errors
    • HTML error - a violation of the rules for writing HTML tags
    • CSS error - a violation of the rules for writing CSS rules
    • syntax error - a statement or expression that violates the syntax rules of the language
    • runtime error - when the interpreter attempts to execute the instructions, an error occurs causing the script to crash; for example, attempting to divide by 0
    • logic error - a mistake in your algorithm; for example, dividing when you should have multiplied

Browser Support for Debugging

Whenever the JavaScript interpreter in the browser encounters a syntax or runtime error while executing your script, it will notify you by producing an error message. It is extremely important and useful to study these error messages when debugging your script. You may need to adjust your browser's settings to see these messages. How you do this depends on the particular browser you are using, as follows:

Chrome

To display the JavaScript Console in Chrome:

Click the Chrome menu (in upper right area of browser window) > More tools > Developer tools > Console. <<<-- Please use this when debugging your programs

For more information about Chrome Developer Tools:

  • Documentation: https://developer.chrome.com/devtools/index
  • CodeSchool has a free course on Chrome DevTools at https://www.codeschool.com/courses/discover-devtools

Firefox

To display the Error Console click Tools > Web Developer > Error Console.

Internet Explorer (aka IE)

To display the various JavaScript options click Tools > Internet Options > Advanced. Also, look for some kind of icon in the status bar at the bottom of your browser window. You may need to click or double-click this to bring up the error message window.

Safari for Mac

  1. Click on Safari in the Menu Bar and select Preferences.
  2. Go to the Advanced tab and check the checkbox 'Show Develop menu in menu bar'.
  3. Close out Preferences.
  4. Click on Develop in the Menu Bar and select Show Error Console.

Errors in Your HTML Code

  • Validate your HTML code before adding scripts and after you add scripts and more HTML.
  • Your HTML editor may have a built-in validator.
  • Or use The W3C MarkUp Validation Service.
  • Common HTML Validation Problems contributed by Christopher Kilian

Errors in Your CSS Code

  • As with HTML, you will find the W3C CSS Validation Service very helpful in finding and correcting your CSS errors.
  • http://jigsaw.w3.org/css-validator/

Syntax Errors

  • violation of the rules of valid/legal statement formation
  • error messages need to be considered carefully; may be hard to understand
  • common errors: missing closing }, ), ", '
  • editor with syntax coloring very helpful; See kengeddes.com/cs81/references.html#JavaScriptIDEs
  • javascript pseudo-protocol
    • enter in Location bar
    • javascript: statement
    • can be bookmarked
  • check the following as you write and review your code:
    • closing }
    • closing )
    • closing "
    • closing '
    • function call matches function name exactly
    • arguments match parameters in number, order, and type
    • watch case on object, property, and methods names; most are initial lowercase; exceptions:
      • Core JavaScript Objects: Array, Boolean, Date, Function, Math, Number, Object, RegExp, String
    • check that names of HTML objects script match those in the HTML
  • Common Syntax Error Messages:
    • expected identifier
    • missing variable name
    • missing (...
    • missing )...
    • missing ; after for-loop condition
    • missing ; after for-loop initializer
    • expected ;
    • missing { before function body
    • missing { after function body
    • missing } in compound statement
    • expected }
    • expected {
    • nested comment
    • syntax error
    • test for equality (==) mistyped as assignment (=)? Assuming equality test
    • unterminated string literal
  • When you test your page, your browser may use a cached copy, so you may need to clear your browser's cache.
