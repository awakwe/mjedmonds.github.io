MySQL Basics

Here is an introduction to MySQL including installation suggestions, overall client/server architecture, related commands, MySQL commands, examples, etc.

In the latter part of our course, we will begin our study of the MySQL Database Management System (DBMS). Then we will write PHP scripts that connect to a MySQL server. Thus, we will be creating 3-tier web apps consisting of the following three tiers:

  1. front end - HTML and CSS running in the browser
  2. middle tier - PHP scripts running on the web server
  3. information tier - SQL statements executed by the MySQL server

MySQL Servers

In this course, you will be using two different MySQL servers:

  1. The one on your machine:
    • It is usually easiest to create and test software on your local machine. You create and manipulate MySQL databases using the MySQL system installed locally. This will be the MySQL that is part of the LAMP stack (such as WampServer) you installed on your machine.
  2. The one on ciswebs.smc.edu:
    • A MySQL server is installed and running on our ciswebs host machine (CLASSWEBS). If you run a PHP script on ciswebs and it connects to the MySQL server on 'localhost', it connects to this MySQL server.
    • Note: You do not have permission to CREATE DATABASE on ciswebs. For your work in this course, you will use databases that have already been created.

Installing MySQL on Windows

Use one of the following two methods to install MySQL on your local machine.

Method 1. Install WampServer 2.0 (really easy/recommended)

By far the easiest way to get MySQL installed on your Windows computer is to install the WampServer. See Install a LAMP stack for more information.

Method 2. Install MySQL (much harder)

Note: By default, a MySQL server "listens" for requests on port 3306. If you want to run both the WampServer and a second MySQL server at the same time (probably not a good idea), be sure to configure them with different port numbers. For more info, see the Install a LAMP stack page.

You can download, install, and configure MySQL on your system for free. Here is how the process went for me on 02/05/13:

  1. Click on www.mysql.com.
  2. Click the Downloads (GA) tab.
  3. Click MySQL Community Server.
  4. Select the Windows platform.
  5. On the Windows (x86, 32-bit), MSI Installer line, click Download.
  6. Skip the login prompts and click No thanks, just start my download.
  7. Save the mysql-5.5.30-win32.msi file on your system. When the download is complete, the MySQL Server 5.5 Setup Wizard should start automatically:
    1. When prompted for Setup Type, select Complete.
  8. When the MySQL Server 5.5 Setup has finished, the MySQL Instance Configuration Wizard should run:
    1. When prompted for configuration type:
      • If you are upgrading an existing installation of MySQL, select Detailed Configuration.
      • If this is a new installation, select Standard Configuration.
    2. Server type: Developer Machine
    3. Database usage: Multifunctional Database
    4. InnoDB Tablespace Settings: use the defaults
    5. Approximate number of concurrent connections to the server: Manual Setting: 5
    6. Networking options: Enable TCP/IP Networking; Port Number: 3306
    7. Server SQL mode: Enable Strict Mode
    8. Default character set: Standard Character Set
    9. Windows options: Install As Windows Service
      1. Service Name: MySQL5.5
      2. Uncheck the Launch the MySQL Server automatically box.
    10. Check the Include Bin Directory in Windows PATH box.
    11. Check the Modify Security Settings box.
      1. Set a root password.
      2. Uncheck the Enable root access from remote machines.
    12. Uncheck the Create an Anonymous Account box.

Adding the mysql\bin folder to the path

The Setup Wizard for MySQL 5.5 will set the path for you automatically. Or, you can do this manually as follows:

Click Start, click Control Panel, double-click System, click the Advanced tab, click the Environment Variables button. Then, in the System variables area, click Path. Click the Edit button. Go to the end of the path string and put a semicolon (;) followed by the path to your mysql\bin\ folder (for example c:\Program Files\MySQL\MySQL Server 5.5\bin)

Client / Server Architecture of MySQL

Note: If you install the WampServer (recommended), you can easily do many of the tasks listed below using the WampServer menu. So, use the menu and

Work smarter, not harder. - Dan Hurley

MySQL has a client/server architecture. Thus, it consists of two parts:
1) a server/daemon (mysqld.exe), and
2) a client/monitor (mysql.exe)

The MySQL server/daemon

The MySQL server can be started and stopped in either of two ways:
1) You can run the server as an OS service, or
2) You can start and stop the server from the Windows Command Prompt.

Running the MySQL server as a service

To install the MySQL server as a service:

1. Start a Windows Command Prompt.

2. mysqld -install

To start the MySQL service:

1. Open the Services window (Control Panel -> Administrative Tools -> Services).

2. Select MySql.

3. Click the Start Service button on the toolbar at the top of the window.

To stop the MySQL service:

1. Open the Services window (Control Panel -> Administrative Tools -> Services).

2. Select MySql.

3. Click the Stop Service button on the toolbar at the top of the window.

To remove the MySQL service:

1. Stop the MySQL service (see above).

2. At the Windows Command Prompt, enter

            mysqld --remove

Starting and stopping the MySQL server from the command prompt

To start the MySQL server:

At the Windows Command Prompt, enter

            mysqld

To stop the MySQL server:

I will assume here that a password has been set for the root user (see below).

1. Start another Command Prompt and enter

            mysqladmin -u root shutdown -p

2. At the Enter password: prompt, enter the password for the root user

Miscellaneous

At the Windows Command Prompt, enter

The MySQL client/monitor

A MySQL client application (aka the MySQL monitor) with a command line interface can be started from a Windows Command Prompt.

Open a Command Prompt window in Windows 8 as follows:
Apps by name > Windows System > Command Prompt

You will now see the Windows command prompt. This is a Command-Line Interface (CLI). When working with a CLI, enter the command, then press the Enter key to run the command.

From the directory where the mysql.exe file is (for example, C:\wamp2.5\bin\mysql\mysql5.6.17\bin) connect to the MySQL server by entering the following at the Command Prompt.

mysql -h localhost -u root -p
Press Enter the the password prompt.

You should now see the mysql> prompt. You can enter MySQL commands and SQL statements at the mysql> prompt.

To get MySQL client help:

To start the MySQL client program:

If you have set the PATH environment variable (see above), you can start the monitor at the Windows Command Prompt as follows. Enter

            mysql

This will start the MySQL client and connect to the MySQL server. Since you have not specified the -h option, the client will connect to the MySQL server running on the localhost, i.e., the same computer as the client. Since you have not specified the -u option, the client will connect as the "anonymous" user. This is insecure. You should delete the anonymous user.

To delete the anonymous user:

        use mysql;

    delete from user where User = '';  Those are two adjacent apostrophes

    quit;

    mysqladmin reload                  This reloads the grant tables into the MySQL server

To connect to MySQL as the root user:

At the Windows Command Prompt, enter

            mysql -u root

This connects as the "root" user. The root user has full administrative privileges. At this point, no password has been set for the root user. This is insecure. You should set a password for the root user.

To set a password for the root user:

1. First exit from the MySQL client program. At the mysql prompt, enter

            quit;

2. At the Windows Command Prompt, enter

            mysqladmin -u root password password_of_your_choice

To create user cs85 with full privileges and a password:

The MySQL grant command allows you to create users, set privileges, and set passwords.

1. First connect as the root user. At the Windows Command Prompt, enter

            mysql -u root -p

2. At the Enter password: prompt, enter the password for the root user

3. At the mysql prompt, enter

       grant all on *.* to cs85 identified by 'password_for_cs85' with grant option;

List of all MySQL commands:

   (Commands must appear first on line and end with ';')

help    (\h)    Display this help.

?       (\?)    Synonym for `help'.

clear   (\c)    Clear command.

connect (\r)    Reconnect to the server. Optional arguments a

ego     (\G)    Send command to mysql server, display result

exit    (\q)    Exit mysql. Same as quit.

go      (\g)    Send command to mysql server.

notee   (\t)    Don't write into outfile.

print   (\p)    Print current command.

prompt  (\R)    Change your mysql prompt.

quit    (\q)    Quit mysql.

rehash  (\#)    Rebuild completion hash.

source  (\.)    Execute a SQL script file. Takes a file name

status  (\s)    Get status information from the server.

tee     (\T)    Set outfile [to_outfile]. Append everything i

use     (\u)    Use another database. Takes database name as

The MySQL source command allows you to run a SQL script file, a text file you can create with a text editor such as Notepad that contains SQL statements.
